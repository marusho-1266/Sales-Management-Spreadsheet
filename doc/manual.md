## 1. システム概要

### 1.1 本システムについて

EC管理システムは、ECサイト運営における以下の業務を効率化するためのGoogleスプレッドシートベースの管理ツールです。

- **在庫管理**: 商品情報の登録・管理、仕入れ元の管理
- **売上管理**: 注文データの記録（手動入力・Joom API連携）
- **利益計算**: 仕入原価・送料・手数料を考慮した事前利益シミュレーション
- **データ出力**: Joom出品用CSVの自動生成
- **価格監視**: 仕入れ価格・販売価格の変動履歴管理と通知
- **自動在庫チェック**: ECサイトからの価格・在庫状況のスクレイピング

<div style="page-break-after: always;"></div>

### 1.2 技術基盤

| 項目 | 技術 |
|------|------|
| 基盤 | Googleスプレッドシート |
| 自動化 | Google Apps Script（GAS） |
| 在庫・価格取得 | InventoryScraper（Selenium） |
| EC連携 | Joom Merchant API v3 |

### 1.3 対象ユーザー

- **管理者**: 全機能利用可能（API設定、シート初期化含む）
- **一般スタッフ**: 商品管理、売上入力、データ出力などの日常業務

### 1.4 シート構成の全体像

本システムは以下のシートで構成されています。初期セットアップは引き渡し時に完了済みです。

| シート名 | 説明 |
|----------|------|
| 在庫管理 | 商品情報・在庫・価格管理（メインシート） |
| 売上管理 | 注文データ・顧客情報・配送管理 |
| 設定 | システム全体の設定値（API認証情報含む） |
| 仕入れ元マスター | 仕入れ先サイトの情報と判定ルール |
| 価格履歴 | 仕入れ・販売価格の変動履歴 |
| 利益計算 | 商品別の事前利益シミュレーション |
| Joom送料マスタ | 配送方法の基本情報 |
| 関税率マスタ | カテゴリ×地域別の関税率 |
| Joom手数料マスタ | カテゴリ×地域別のJoom手数料率 |
| 重量別送料マスタ | 配送方法×重量×地域の送料テーブル |
| 繁忙期料金マスタ | 配送方法×地域×期間別の繁忙期追加料金（開始日・終了日で期間指定可） |

## 3. 商品管理

### 3.1 新商品追加

新しい商品を在庫管理シートに登録します。

**操作手順**

1. メニュー「**EC管理システム**」→「**商品管理**」→「**新商品追加**」を選択
2. 入力フォームが表示される
3. 各項目を入力する

**入力項目一覧**

| 項目 | 必須 | 説明 |
|------|------|------|
| 商品ID | 自動 | 自動採番（変更可） |
| 商品名 | ○ | 商品の名称 |
| JoomID | - | Joom製品ID（Joom出品後に設定） |
| SKU | - | 商品管理コード |
| ASIN | - | Amazon商品コード |
| 仕入れ元 | ○ | 仕入れ先サイト名（ドロップダウン） |
| 仕入れ元URL | ○ | 商品ページのURL |
| 仕入れ価格(円) | ○ | 仕入れ価格 |
| 販売価格(円) | - | 販売価格 |
| 販売価格(USD) | - | USD建ての販売価格 |
| 重量(g) | ○ | 商品重量 |
| 高さ/長さ/幅(cm) | ○ | 寸法（容積重量計算用） |
| 商品カテゴリー | ○ | 基本的には「一般」を設定 |
| 商品説明 | - | Joom出品用の説明文 |
| メイン画像URL | - | 商品画像のURL |
| 通貨 | - | 通貨（デフォルト: JPY） |
| 配送価格(円) | - | 配送価格 |
| 在庫数量 | ○ | 在庫の個数 |
| 在庫ステータス | ○ | 在庫あり/売り切れ |
| 備考・メモ | - | 自由記入欄 |

4. 「保存」ボタンをクリック
5. 保存成功メッセージが表示され、フォームが閉じる

**補足**
- 商品IDは自動採番されますが、任意のIDに変更可能です
- 新商品追加後、利益計算シートの商品IDドロップダウンにも自動反映されます
- 在庫管理シートの利益列（Y列）に自動計算式が設定されます

### 3.2 商品削除

在庫管理シートから商品を削除します。

**操作手順**

1. メニュー「**EC管理システム**」→「**商品管理**」→「**商品削除**」を選択
2. 商品ID入力ダイアログが表示される
3. 削除したい商品のIDを入力
4. 確認ダイアログが表示される
   - 売上履歴がある場合は警告メッセージが表示される
5. 「OK」で削除を実行

**注意事項**
- 削除は物理削除（行ごと完全に削除）です。元に戻せません
- 売上管理シートの注文データは削除されません（履歴として残ります）
- 利益計算シートで該当商品を選択中の場合、選択がクリアされます

## 4. 売上管理

売上データの登録は、**手動入力**と**Joom API連携**の2つの方法があります。

### 4.1 注文データ手動入力

**操作手順**

1. メニュー「**EC管理システム**」→「**売上管理**」→「**注文データ入力**」を選択
2. 入力フォームが表示される

**入力項目一覧**

| 項目 | 必須 | 説明 |
|------|------|------|
| 注文ID | ○ | 注文の識別番号（任意の文字列） |
| 注文日 | ○ | 注文日（日付選択） |
| 商品 | ○ | 商品選択モーダルから選択 |
| 数量 | ○ | 注文数量 |
| 販売価格(円) | ○ | 実際の販売価格 |
| 仕入れ価格(円) | ○ | 仕入れ価格（在庫管理から自動取得） |
| 送料(円) | - | 配送料（デフォルト: 0） |
| 手数料(円) | - | 手数料（デフォルト: 0） |
| 付加価値税(円) | - | VAT（デフォルト: 0） |
| 返金額(円) | - | 返金額（デフォルト: 0） |

3. 商品選択は「商品を選択」ボタンをクリックして、検索付きモーダルから選択
4. 入力内容のプレビューで純利益が自動計算される
   - **純利益 = 販売価格 - (仕入れ価格 + 送料 + 手数料 + 付加価値税 + 返金額)**
5. 「保存」ボタンをクリック

<div style="page-break-after: always;"></div>

**補足**
- 売上管理シートのK列（純利益）には計算式 `=H-(I+J+P+Q+R)` が設定されます
- 保存時に在庫管理シートの在庫数量が自動的に減算されます
- 在庫が0になった場合、在庫ステータスが「売り切れ」に自動更新されます

### 4.2 Joom注文取得（API連携）

Joom APIから注文データを自動取得し、売上管理シートに登録します。

> 事前に「Joom API設定」でトークンを取得しておく必要があります（[9. Joom API設定](#9-joom-api設定)参照）

#### 4.2.1 最新注文を取得

前回連携以降の新しい注文を取得します。

1. メニュー「**EC管理システム**」→「**売上管理**」→「**Joom注文取得**」→「**最新注文を取得**」
2. 前回連携時刻以降の注文が自動取得される
3. 処理結果（取得件数、登録件数、スキップ件数）が表示される

#### 4.2.2 日時範囲で取得

指定した日時範囲の注文を取得します。

1. メニュー「**EC管理システム**」→「**売上管理**」→「**Joom注文取得**」→「**日時範囲で取得**」
2. 開始日時と終了日時を入力
3. 指定範囲の注文が取得・登録される

#### 4.2.3 特定注文を取得

指定した注文IDの注文を取得します。

1. メニュー「**EC管理システム**」→「**売上管理**」→「**Joom注文取得**」→「**特定注文を取得**」
2. 注文IDを入力
3. 該当注文が取得・登録される

#### 4.2.4 取得状況確認

前回連携時刻などの取得状況を確認します。

1. メニュー「**EC管理システム**」→「**売上管理**」→「**Joom注文取得**」→「**取得状況確認**」

**Joom連携時の自動処理**
- 重複チェック: 既存の注文IDと照合し、重複登録を防止
- 商品情報補完: 商品ID（SKU）で在庫管理シートから商品名・仕入れ価格等を自動取得
- 為替変換: Joom APIの価格を設定シートの為替レートで円換算
- 在庫減算: 注文数量分の在庫を自動減算
- 純利益計算: K列に計算式 `=H-(I+J+P+Q+R)` を自動設定

## 5. データ出力（Joom用CSV）

在庫管理シートのデータからJoom出品用CSVファイルを生成します。

### 5.1 CSV出力手順

1. メニュー「**EC管理システム**」→「**データ出力**」→「**Joom用CSV出力**」を選択
2. **未連携商品**（Joom連携ステータスが「未連携」の商品）のみが出力対象
3. バリデーションが実行され、必須項目に不備がある場合はエラーレポートが表示される
4. CSVファイルが生成され、ダウンロードされる
5. 出力完了後、対象商品の連携ステータスが自動的に「連携済み」に更新される

### 5.2 CSV出力フィールド

| フィールド | 必須 | 説明 | 変換 |
|------------|------|------|------|
| Product SKU | ○ | 商品管理コード | - |
| Name | ○ | 商品名 | - |
| Description | ○ | 商品説明 | - |
| Product Main Image URL | ○ | メイン画像URL | - |
| Store ID | ○ | ストアID（設定シートから取得） | - |
| Shipping Weight | ○ | 重量 | g → kg変換 |
| Price | ○ | 販売価格 | 円 → 指定通貨変換 |
| Currency | ○ | 通貨 | - |
| Inventory | ○ | 在庫数量 | - |
| Shipping Price | ○ | 配送価格 | - |
| Brand | - | ブランド | - |
| Search Tags | - | 検索タグ | - |
| Dangerous Kind | - | 危険物種類 | - |
| Suggested Category ID | - | カテゴリID | - |

<div style="page-break-after: always;"></div>

### 5.3 出力条件

- 在庫数量が0または未設定の商品は**自動除外**
- 連携ステータスが「未連携」の商品のみ出力（メニューから実行時）
- 必須項目（SKU、商品名、説明、画像URL、重量、価格）が未入力の場合はバリデーションエラー

## 6. 利益計算

利益計算シートは、1商品ずつ利益をシミュレーションするための専用シートです。

### 6.1 基本的な使い方

#### 商品データの読み込み

1. 利益計算シートのB2セルで、在庫管理シートの商品IDをドロップダウンから選択
2. 選択すると在庫管理シートから以下のデータが自動読み込みされる：
   - 商品名、SKU、仕入URL、仕入元
   - 販売価格、仕入価格、重量、寸法
3. または、メニュー「**EC管理システム**」→「**利益計算**」→「**商品データ読み込み**」で手動読み込み

#### 利益計算のしくみ

利益計算シートでは以下の計算が自動で行われます。

```
利益額 = 販売価格 - 仕入原価 - 送料 - Joom手数料 - サーチャージ - 繁忙期料金
利益率 = 利益額 / 販売価格
```

各費用はマスタシートから自動取得されます。

| 計算要素 | 参照先 | 決定条件 |
|----------|--------|----------|
| 送料 | 重量別送料マスタ | 配送方法 × 重量 × 配送地帯 |
| Joom手数料 | Joom手数料マスタ | 商品カテゴリー × 配送地帯 |
| サーチャージ | Joom送料マスタ | 配送方法 |
| 繁忙期料金 | 繁忙期料金マスタ | 配送方法 × 配送地帯。開始日・終了日が設定されている場合は当日が期間内のときのみ適用。空の場合は常に適用。 |

<div style="page-break-after: always;"></div>

#### 容積重量チェックセクション

| セル | 項目 | 説明 |
|------|------|------|

| B36 | USD換算販売額 | 販売価格 / 最終為替レート |

### 6.3 利益計算データの在庫管理への反映

利益計算シートで算出した結果を在庫管理シートに書き戻すことができます。

1. メニュー「**EC管理システム**」→「**利益計算**」→「**利益計算データを在庫管理へ反映**」
2. 以下の項目が在庫管理シートに反映される：
   - 販売価格、仕入れ価格、重量、寸法
   - Joom手数料、サーチャージ、繁忙期料金
   - 利益額、最終為替レート、USD換算販売額

### 6.4 参照式更新

マスタシートの構成が変更された場合に、利益計算シートの参照式を再設定します。

1. メニュー「**EC管理システム**」→「**利益計算**」→「**参照式更新**」

### 6.5 利益計算シート初期化

利益計算シートを初期状態に戻します。

1. メニュー「**EC管理システム**」→「**利益計算**」→「**利益計算シート初期化**」

**補足**：実行時に送料セル（E16）で #ERROR が発生する場合は、一度セルをダブルクリックすると解消されます。

## 7. 価格履歴

### 7.1 価格履歴の概要

価格履歴シートは、各商品の仕入れ価格と販売価格の変動を1商品1行で管理します。

**記録される項目**
- 現在の仕入れ価格 / 前回の仕入れ価格
- 仕入れ価格の変動額・変動率
- 現在の販売価格 / 前回の販売価格
- 販売価格の変動額・変動率
- 変動回数
- 最終更新日時

### 7.2 価格履歴の更新方法

価格履歴は以下のタイミングで自動更新されます。

| タイミング | 条件 |
|------------|------|
| 在庫管理シートの手動編集 | 仕入れ価格（H列）または販売価格（I列）を変更した場合（トリガー設定済み） |

手動で一括同期することもできます：

1. メニュー「**EC管理システム**」→「**価格履歴**」→「**価格履歴同期**」

### 7.3 価格変動通知の設定

価格に変動があった場合にメールで通知を受け取ることができます。

#### メールアドレスの設定

1. メニュー「**EC管理システム**」→「**価格履歴**」→「**メールアドレス設定**」
2. 通知先のメールアドレスを入力

1. メニュー「**EC管理システム**」→「**価格履歴**」→「**通知有効化設定**」
2. 有効/無効を選択

**通知メールの内容**
- 商品名
- 変更前の価格
- 変更後の価格
- 変動率

## 10. ウェブスクレイピング（在庫・価格自動取得）

### 10.1 概要

InventoryScraper.exe を使用して、仕入れ先ECサイトから商品の価格と在庫状況を自動取得し、在庫管理シートに反映します。

**対応サイト**
- Amazon
- メルカリ / メルカリShops
- Yahoo!ショッピング
- ヤフオク
- 楽天
- その他（仕入れ元マスターで設定追加可能）

### 10.2 配布パッケージのセットアップ（ZIP展開とフォルダ設定）

スクレイピング用のexeを含むフォルダはZIPファイルとして納品されます。以下でZIP展開からセットアップまでの手順を説明します。

#### 必要な環境

- **Windows 10以上**
- **Google Chrome**（インストール済み）
- **インターネット接続**
- **Googleアカウント**（スプレッドシートへのアクセス権限が必要）

#### 手順1: ZIPファイルの展開

1. 配布されたZIPファイル（例: `inventory_scraper.zip`）を任意の場所に保存
2. ZIPファイルを右クリック → 「すべて展開」を選択
3. 展開先を指定（推奨: `C:\Users\[ユーザー名]\InventoryScraper` または `C:\Program Files\InventoryScraper`）
4. 「抽出」をクリックして展開を完了

#### 手順2: フォルダ構成の確認

展開後のフォルダには以下のファイル・フォルダが含まれています。

```
inventory_scraper/
├── InventoryScraper.exe    # メイン実行ファイル
├── .env                    # 環境設定ファイル（要編集）
├── config/                 # 設定フォルダ
├── readme.txt              # 簡易説明
├── data/                   # 実行時に自動作成（CSV出力先）
└── logs/                   # 実行時に自動作成（ログ保存先）
```

#### 手順3: 環境設定ファイル（.env）の編集

1. フォルダ内の **`.env`** ファイルをメモ帳などで開く
   - 表示されない場合: エクスプローラーで「表示」→「隠しファイル」にチェックを入れる
   - または `.env.example` が存在する場合はコピーして `.env` にリネーム

2. 以下の項目を自分の環境に合わせて編集する

| 設定項目 | 説明 | 取得方法 |
|----------|------|----------|
| CHROME_PROFILE_PATH | Chromeプロファイルパス | 通常は `C:\Users\[ユーザー名]\AppData\Local\Google\Chrome\User Data`（`%LOCALAPPDATA%\Google\Chrome\User Data` で確認可） |

3. ファイルを保存して閉じる

### 10.3 スクレイピングの仕組み

```
[Pythonスクレイパー]
    │
    ├── 1. スプレッドシートから在庫データをCSVダウンロード
    ├── 2. 各商品の仕入れ元URLにアクセス
    ├── 3. 価格・在庫状況を取得
    ├── 4. 結果CSVを生成
    └── 5. GAS WebアプリにPOST送信 → スプレッドシート更新
```

### 10.4 スクレイピングの実行方法

スクレイピングは **InventoryScraper.exe** を実行して行います。

**操作手順**

1. **`InventoryScraper.exe`** をダブルクリックして実行
2. コンソールウィンドウが表示され、処理が開始される
3. 処理が完了すると、スプレッドシートが自動的に更新される

### 10.5 スクレイピング結果の反映

スクレイピング結果は以下の項目が在庫管理シートに自動反映されます。

| 項目 | 説明 |
|------|------|
| 仕入れ価格 | 仕入れ元サイトから取得した現在価格 |
| 在庫ステータス | 在庫あり/売り切れ |
| 最終更新日時 | 更新日時 |

仕入れ価格が変更された場合、価格履歴も自動更新されます。

### 10.6 仕入れ元マスターの設定

新しい仕入れ先サイトを追加する場合は、仕入れ元マスターシートに登録します。

| 項目 | 説明 |
|------|------|
| サイト名 | 仕入れ先サイト名 |
| URLパターン | ベースURL（カンマ区切り） |
| 価格セレクタ | 価格表示要素のCSSセレクタ |
| 価格除外セレクタ | 除外する要素のセレクタ |
| 在庫セレクタ | 在庫表示要素のセレクタ |
| 在庫ありキーワード | 在庫あり時の検出文字列 |
| 売り切れキーワード | 売り切れ時の検出文字列 |
| 手数料率(%) | サイトの手数料率 |
| アクセス間隔(秒) | リクエスト間隔 |
| 有効フラグ | 有効/無効 |

## 12. トラブルシューティング

### 12.1 一般的な問題

#### メニューが表示されない

**原因**: スクリプトの権限が未承認、またはスクリプトにエラーがある

**対処法**:
1. ページを再読み込み（F5）
2. 「拡張機能」→「Apps Script」でエラーがないか確認
3. `setupCustomMenu` 関数を手動実行して権限を承認

#### シートの構造が壊れた

**対処法**:
1. メニュー「システム管理・設定」→「個別シート初期化・再作成」で該当シートのみ再初期化
2. 全体が壊れた場合は「全シート初期化」を実行（データは失われます）

### 12.2 利益計算の問題

#### 商品IDドロップダウンに新商品が表示されない

**原因**: ドロップダウンの範囲が古い

**対処法**:
- 新商品を追加すれば自動反映されます
- 反映されない場合は「利益計算」→「利益計算シート初期化」を実行

#### 利益計算の値が正しくない

**確認事項**:
1. マスタシートにデータが正しく入力されているか
2. 配送方法・配送地帯が正しく選択されているか
3. 重量が正しく入力されているか
4. 「参照式更新」を実行して参照式を再設定

### 12.3 Joom API連携の問題

#### 認証エラーが発生する

**対処法**:
1. 「JoomAPI設定」→「トークンステータス確認」でトークン状態を確認
2. トークンが期限切れの場合は「トークンリフレッシュ」を実行
3. リフレッシュが失敗する場合は「トークンリセット」→「トークン取得」で再認証

#### 注文データが重複する

**説明**: Joom注文連携には注文IDベースの重複チェックが実装されています。通常、同じ注文が二重に登録されることはありません。

**万が一重複した場合**:
1. 売上管理シートで該当の重複行を手動削除
2. 在庫数量を手動で修正

#### 同期エラーが発生する

**確認事項**:
1. インターネット接続を確認
2. Joom APIのレート制限（2000 rpm）に達していないか確認
3. 「同期状況確認」でエラーメッセージを確認
4. 必要に応じてトークンをリフレッシュ

### 12.4 価格履歴の問題

#### 価格を変更しても履歴が更新されない

**確認事項**:
1. `onInventorySheetEdit` トリガーが設定されているか（[付録B](#付録b-トリガー設定手順)参照）
2. 変更したセルが仕入れ価格（H列）または販売価格（I列）であるか
3. 対象行に商品IDが設定されているか
4. 手動同期: 「価格履歴」→「価格履歴同期」を実行

### 12.5 スクレイピングの問題

#### スクレイピングが失敗する

**確認事項**:
1. インターネット接続を確認
2. Chromeが最新版か確認
3. 対象サイトのHTML構造が変更されていないか確認
4. アクセスがブロックされていないか確認
5. ログファイル（`logs`フォルダ内）を確認

#### スプレッドシートに結果が反映されない

**確認事項**:
1. `.env`の`GAS_WEB_APP_URL`が正しいか確認
2. GAS Webアプリが最新バージョンでデプロイされているか確認
3. Apps Scriptの実行ログでエラーを確認

## 13. 制約事項・注意点

### 13.1 Google Apps Script（GAS）の制約

| 制約 | 上限 |
|------|------|
| スクリプト実行時間 | 6分/実行 |
| 1日あたりのAPI呼び出し | 制限あり |
| メール送信数 | 1日あたりの上限あり |

### 13.2 スプレッドシートの制約

| 制約 | 上限 |
|------|------|
| セル数上限 | 500万セル |
| 同時編集 | 複数ユーザー対応（競合に注意） |
| 計算負荷 | 大量データ時に動作速度低下の可能性 |

### 13.3 Joom APIの制約

| 制約 | 上限 |
|------|------|
| レート制限 | 2000 rpm（1分あたりのリクエスト数） |
| トークン有効期限 | 定期的なリフレッシュが必要 |
| データ形式 | JSON形式 |

<div style="page-break-after: always;"></div>

### 13.4 スクレイピングの制約

- ECサイトの仕様変更により、価格・在庫判定が正しく動作しなくなる場合があります
- 過度なアクセスはIP制限のリスクがあります（アクセス間隔を設定してください）
- Amazon は利用規約によりスクレイピングに制限があります

### 13.5 利益計算について

- **在庫管理シートY列（商品単位）の利益**: 仕入原価・送料・Joom手数料・サーチャージ・繁忙期料金を考慮。**返金額は含まない**
- **売上管理シートK列（注文単位）の利益**: 仕入原価・送料・手数料・VAT・返金額を含む。注文ごとの実績利益
- 返金額は注文に対する個別対応のため、売上管理シートで注文単位に管理し、商品単位の利益計算には含めません

## 付録A. 初期セットアップ手順

> 本付録は、システムの再セットアップやシート構造の再構築が必要になった場合の参照用です。通常運用では実行不要です。

### A.1 全シートの初期化

1. スプレッドシートを開く
2. メニュー「**EC管理システム**」→「**システム管理・設定**」→「**全シート初期化**」を選択
3. 確認ダイアログで「OK」をクリック
4. 全シート（在庫管理、売上管理、設定、仕入れ元マスター、価格履歴、利益計算、各マスタシート）が作成される

### A.2 設定シートの初期値

初期化後、設定シートに以下の初期値が設定されます。必要に応じて変更してください。

| 設定項目 | 初期値 | 説明 |
|----------|--------|------|
| ストアID | （空） | JoomストアIDを入力 |
| デフォルト通貨 | JPY | 基本通貨 |
| デフォルト配送価格 | 0 | 配送価格のデフォルト値 |
| 為替レート USD/JPY | 150 | 米ドル→円の為替レート |
| 価格変動通知メールアドレス | （空） | 通知先メール |
| 価格変動通知有効化 | FALSE | 通知の有効/無効 |

### A.3 GAS Webアプリのデプロイ

スクレイピング結果をスプレッドシートに反映するためのWebアプリデプロイ手順です。

1. 「**拡張機能**」→「**Apps Script**」でスクリプトエディタを開く
2. 「**デプロイ**」→「**デプロイを管理**」→「**新しいデプロイ**」
3. 種類：「**ウェブアプリ**」を選択
4. 設定：
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員（匿名ユーザーを含む）**
5. 「**デプロイ**」をクリック
6. 表示されたWebアプリURLをコピーし、スクレイパーの `.env` ファイルに設定

> コードを更新した場合は「デプロイを管理」→「新しいバージョンを保存」を実行してください。

## 付録B. トリガー設定手順

> 本付録は、トリガーの再設定が必要になった場合の参照用です。引き渡し時に設定済みのため、通常は変更不要です。

### B.1 価格変動自動検知トリガー

在庫管理シートで価格を手動変更した際に、自動的に価格履歴へ反映させるためのトリガーです。

1. 「**拡張機能**」→「**Apps Script**」を選択
2. 左側メニューから「**トリガー**」をクリック
3. 「**+ トリガーを追加**」ボタンをクリック
4. 以下の設定を入力：

| 項目 | 設定値 |
|------|--------|
| 実行する関数 | `onInventorySheetEdit` |
| イベントソース | スプレッドシートから |
| イベントタイプ | 編集時 |
| エラー通知設定 | 通知する |

5. 「保存」をクリック
6. 初回は権限の承認が求められるため、画面に従って許可する

### B.2 権限の承認手順

トリガー設定時やスクリプト初回実行時に権限の承認が求められます。

1. 「**権限を確認**」をクリック
2. Googleアカウントを選択
3. 「**詳細**」→「**（プロジェクト名）に移動（安全ではありません）**」をクリック
4. 「**許可**」をクリック

### B.3 トリガーの確認方法

設定済みのトリガーを確認する場合：

1. 「**拡張機能**」→「**Apps Script**」
2. 左側メニューの「**トリガー**」をクリック
3. 登録済みトリガーの一覧が表示される
